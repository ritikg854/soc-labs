# Endpoint Alert Review

## Objective

The purpose of this case study is to practice documenting how an endpoint-related alert may be reviewed during initial security triage.

## Scenario

In this exercise, I reviewed an endpoint alert scenario to understand how an analyst may assess context, potential risk, and the need for further investigation or escalation.

## Investigation Focus

- Type of endpoint alert
- User and device context
- Possible benign explanations
- Signs of suspicious behaviour
- Alert severity and escalation considerations

## Analyst Approach

- Review the alert details carefully
- Consider the endpoint and user context
- Assess whether the activity appears expected, low risk, or suspicious
- Note whether additional data would be needed
- Document findings clearly and logically

## Key Learning

This case study helped improve my understanding of practical endpoint alert triage and strengthened my ability to document analyst-style review steps in a structured way.

## Skills Demonstrated

- Endpoint alert review
- Security triage thinking
- Risk-oriented assessment
- Documentation
- Analyst-focused reasoning
