# Failed Logon Investigation

## Objective

The purpose of this case study is to practice identifying and documenting failed logon activity that may be relevant during security monitoring and initial triage.

## Scenario

In this exercise, I reviewed Windows log activity with a focus on failed authentication events. The goal was to understand how repeated failed logons may appear during normal operations, user error, misconfiguration, or potentially suspicious access attempts.

## Investigation Focus

- Failed logon patterns
- Repeated authentication attempts
- Source account review
- Time-based event review
- Distinguishing normal user behaviour from suspicious activity

## Analyst Approach

- Review relevant Windows security events
- Look for repeated failures over a short period
- Check whether the same account appears multiple times
- Consider whether activity may be caused by user error, stale credentials, or unauthorised access attempts
- Document observations clearly for escalation if needed

## Key Learning

This case study helped reinforce the importance of authentication monitoring, event review, and structured investigation notes in security operations work.

## Analyst Takeaway

This case study helped reinforce the importance of structured triage, context-based review, and clear documentation in security operations work.

## Skills Demonstrated

- Basic investigation workflow
- Triage thinking
