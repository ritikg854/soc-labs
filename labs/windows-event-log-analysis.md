# Windows Event Log Analysis Lab

## Objective

The objective of this lab is to practice reviewing Windows event logs to identify potentially suspicious activity, understand common event sources, and build a structured investigation approach relevant to SOC and security analyst roles.

## Scenario

In this lab, I explored Windows event logs to understand how security-relevant activity can appear during normal system usage and potentially suspicious behaviour. The goal was to build familiarity with event visibility, investigation logic, and documentation.

## Key Areas Reviewed

- Windows Security logs
- System logs
- Application logs
- Logon-related events
- Account activity events
- Process and service-related activity
- Signs of failed logon attempts or unusual behaviour

## What I Practiced

- Reviewing Windows Event Viewer logs
- Identifying useful event categories for investigation
- Understanding how log data supports alert triage
- Noting examples of normal vs potentially suspicious activity
- Writing structured observations like a junior SOC analyst

## Outcome

This lab helped strengthen my understanding of endpoint visibility and Windows-based investigation workflows. It also improved my ability to document findings clearly and think in a more analyst-focused way.
