# Common Windows Security Events

## Purpose

This note captures some common types of Windows security-relevant activity that are useful during monitoring, triage, and endpoint investigation.

## Important Areas to Review

- Logon activity
- Failed authentication attempts
- Account lockouts
- Privilege-related activity
- Service activity
- Process-related behaviour
- System changes
- User account events

## Why These Events Matter

Windows event logs can help analysts understand what happened on an endpoint, when it happened, which account was involved, and whether activity appears expected or suspicious.

## Analyst Value

Reviewing these event types helps support:

- initial alert triage
- endpoint investigation
- authentication monitoring
- suspicious activity review
- escalation decisions
- incident documentation

## Learning Outcome

This note helps build a stronger foundation in Windows log awareness and supports later work in SOC analysis, endpoint monitoring, and incident response support.
